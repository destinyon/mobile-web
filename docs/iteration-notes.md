# 迭代说明

## 第 1 轮

- 补齐仓库基础说明与忽略规则。
- 新建 Spring Boot 后端骨架、H2 数据库、初始化数据与核心 API。
- 新建原生微信小程序页面骨架、组件、请求封装和主题样式。

## 第 2 轮

- 完成新闻列表、详情、收藏、点赞、评论、我的收藏、我的评论、我的发布、发布内容与后台接口。
- 补充微信登录、OSS 上传、接口文档和集成测试。
- 将后端搜索查询改为参数化，避免字符串拼接。
- 在微信开发者工具中抓取首页、我的页、新闻详情页截图，确认小程序可加载真实后端数据。

## 第 3 轮

- 修复微信登录 500：拆分 `WxSessionClient`，将微信错误码和网络异常转换为业务响应。
- 修复种子数据自增序列未推进导致首次登录插入用户撞主键的问题。
- 新闻同步改为抓取网易体育羽毛球滚动新闻前 10 页，通过 `POST /api/admin/news/sync` 触发。
- 同步改为按网易新闻 `source_id` 和原文发布时间增量入库；已存在或发布时间不新的新闻会跳过，旧新闻、评论、点赞和收藏数据会保留。
- 网易新闻正文入库前会清洗为安全 HTML，保留正文图片和图片下方说明，封面图不重复放到正文开头，标题按原文完整保存。
- 后端启动时自动同步一次网易新闻，随后默认每 5 分钟同步一次；测试环境关闭自动同步。
- 重写中文乱码文件，修复搜索页 placeholder 属性错误。
- 优化小程序 UI：更贴近原型的浅绿底、黑描边、紧凑新闻卡片、详情页标题栏、个人中心列表式功能区。
- 验证结果：后端 `mvn test -q` 通过；小程序 JS 语法和 JSON 解析检查通过。

## 第 4 轮

- 小程序范围收敛为可演示闭环，不开发独立后台管理前端。
- 首页顶部“发布”按钮移除，改为五栏自定义底部导航：首页、资讯、中间圆形 `+` 发布、社区、我的。
- 新增社区帖子能力：话题、帖子列表、帖子详情、发帖、帖子点赞、帖子收藏、评论与回复。
- 后端新增 `topics`、`posts` 表，并复用 `comments`、`favorites`、`likes` 的 `target_type='POST'` 支持社区互动。
- 发布页改为发布社区帖子，“我的发布”改为展示真实帖子数据。
- 补充种子新闻和种子帖子，避免演示依赖外部新闻同步或空列表。
- 验证结果：后端 `mvn test -q` 通过；微信开发者 MCP 编译通过；MCP 页面数据验证首页、社区、发帖、详情、评论回复、我的发布均可读取真实后端数据，控制台无错误。

## 第 5 轮

- 新增后台管理前端 `web-admin`，提供管理员登录、运营概览、文章管理、热度排行和用户管理。
- 后台文章列表改为读取 `news` 表全量分页数据，不再复用前台只看已发布文章的列表链路。
- 后台文章详情改为只读预览，不增加浏览量；热度排行和用户列表支持点击查看详情。
- 登录页使用项目内羽毛球背景图，概览页增加分类分布、互动热度和发布结构图表。
- 非测试环境默认不执行 `schema.sql`/`data.sql`，避免启动时重建真实数据库；测试环境仍使用 H2 自动初始化。
